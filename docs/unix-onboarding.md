# Unix onboarding notes

## Design

The Unix onboarding path targets Linux and macOS with OpenSSH and `step`.

It intentionally uses a dedicated client root:

```text
~/.config/kaffeerunde-ssh
```

This keeps the onboarding state separate from normal SSH keys and agents.

## Aliases

```text
kaffeerunde-mgmt-01
kaffeerunde-egress-01
kaffeerunde-auth-01
kaffeerunde-ingress-01
kaffeerunde-apps-01
```

Private hosts use `ProxyJump kaffeerunde-mgmt-01`.

## Integrity choices

The client config pins host verification to the repository-provided `known_hosts` file and uses:

```sshconfig
StrictHostKeyChecking yes
UpdateHostKeys no
```

This avoids silently trusting new or replaced host keys.

The `known_hosts` file is generated by reading public host keys over an already-authenticated SSH path, not by blindly accepting unauthenticated `ssh-keyscan` output.

## Empty-home test

`tests/empty-home-unix.sh` uses a temporary `/tmp` home directory and `env -i` so that existing local SSH config, existing agents and exported `KO_*` variables do not make the test pass accidentally.
